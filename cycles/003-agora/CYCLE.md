# Cycle 003 — 아고라 (Agora)

- **기간**: 2026-07-13 ~ 2026-07-13
- **상태**: 완료
- **참여**: 박상현, 다이달로스

## 1. 이전 사이클로부터의 교훈

- Cycle 002: POST 본문이 핸들러의 `input`이 되고, `input`은 untrusted라 check를 거쳐야만 효과·emit에 닿는다 — 왕복 소통의 원형이 드라이버 수준에 이미 있다.
- Cycle 002: `.ail`은 자기가 손대는 공간 안에 산다. cap 최소, det로 되는 일은 det로.
- 유산 정리(HERITAGE.md): 옛 아고라는 "한 문단"의 이름뿐인 기관이었다. 기관은 검증된 가설로만 세운다 — 그래서 아고라를 사이클로 짓는다. 또한 새 폴리스의 아고라는 옛것과 달리 **사용자가 참여자**다 (VISION.md).

## 2. 가설

> 사람이 폴리스에 메시지를 보내면(POST), 그 메시지가 check를 거쳐 아고라 장부에 기록되고 접수 응답이 돌아온다 — 사람↔폴리스의 왕복 소통이 AIL 보증 체계 안에서 성립한다.

## 3. 실험 설계

- `polis/agora.ail`: 메시지 수신(`input`) → check(길이 1~2000자) → 장부(`agora/ledger.txt`)에 연번과 함께 추가 → 접수 번호로 응답. 연번은 `state`(agora_count)로 유지. **순수 det, 0tok.**
- `polis/viewer.ail` 확장: 아고라 장부를 사람의 창에 표시.
- 관찰: (a) check 통과 메시지가 장부에 기록되고 접수 번호가 돌아오는가, (b) 그 기록이 사람의 창(브라우저)에 보이는가, (c) 보증 리포트가 "무엇이 허용되고 무엇이 일어났는지" 보여주는가.
- 알려진 제약: AIL det ops에 시계가 없어 타임스탬프 불가 → 연번으로 대체. (필드 갭으로 기록해둘 것)

## 4. 실험 기록

- `agora.ail` v1 작성, `ail run`으로 창건 발언 [1]이 장부에 기록됨 (fs.write 77자, state.write, 0tok).
- 8766 포트에 서버 기동, POST 발언 [2] 접수 → 사람의 창(8765)에서 표시 확인. **왕복 성립.**
- **사고 발견**: 빈 POST가 "아고라에 기록되었습니다 [3]"으로 응답되는데 장부에는 [3]이 없었다. 원인: check 실패의 격리(quarantine)는 **그 값에서 파생된 가지만** 막는다. 장부 쓰기(msg 파생)는 하네스가 정확히 거부했지만, 응답문과 카운터는 msg에서 파생되지 않아 실행됐다. **하네스는 정직했고, 거짓말은 내 프로그램 구조에 있었다.**
- v2 재설계: 접수 번호와 응답을 msg 파생값으로 묶음 (`len(msg)` → select → delta → num → reply). 이제 check 실패 시 쓰기·카운터·응답이 전부 격리되어 드라이버가 **HTTP 403 (REFUSED by harness)** 를 반환.
- 재검증: 유효 POST → [3] 기록·응답 일치. 빈 POST → 403, 카운터 불변, 장부 불변. (버그 기간에 헛돈 카운터는 수동으로 2로 복구 — state 수리도 기록에 남긴다.)
- 필드 갭: det ops에 시계가 없어 타임스탬프 불가 → 연번으로 대체, `ail ask`로 상류에 기록.
- `polis/.ail-state/`(연번 카운터)는 도시의 상태이므로 깃에 포함 — 다른 맥에서도 연번이 이어진다.

## 5. 결과

**검증됨.** (a) check 통과 발언이 장부에 기록되고 접수 번호가 돌아옴, (b) 기록이 사람의 창에 렌더링됨, (c) 보증 리포트가 cap 3개(장부 읽기/쓰기, 카운터)·검증 2건·효과 2건·0tok을 정확히 보여줌. 거부 경로(403)까지 실측 확인 — 사람↔폴리스 왕복 소통이 보증 체계 안에서 성립한다.

## 6. 교훈

1. **약속은 약속의 대상과 같은 운명이어야 한다.** "기록되었다"는 응답이 기록에서 파생되지 않으면, check 실패 시 프로그램이 구조적으로 거짓말을 한다. 격리는 가지 단위다 — 성공을 알리는 모든 출력을 검사 대상 값에서 파생시킬 것. 이것은 아고라만이 아니라 앞으로 모든 기관 핸들러의 설계 원칙.
2. **거부는 403으로 정직하게.** emit까지 격리되면 드라이버가 REFUSED와 보증 리포트를 반환한다 — 실패를 숨기지 않는 인터페이스가 공짜로 따라온다.
3. state는 도시의 일부다 — 깃으로 관리. 단, 프로그램 버그로 오염된 state의 수리는 수동 개입이 필요했다. 다음에 기관이 늘면 "state 정합성 점검" 도구가 필요할 수 있다.
4. 다음 사이클(004 아르케)에서 처음으로 llm 유닛이 필요하다 — 아고라의 발언을 읽고 의미로 응답하는 존재. 아고라 장부가 이미 있으니 아르케는 태어나자마자 말할 곳이 있다.
