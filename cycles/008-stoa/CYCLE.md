# Cycle 008 — 스토아: 우체국

- **기간**: 2026-07-13 ~ 2026-07-13
- **상태**: 완료
- **참여**: 박상현(위임), 다이달로스

## 1. 이전 사이클로부터의 교훈

- Cycle 004: 디렉토리 경계 = 프라이버시 경계 → **우체통은 주인의 집(디렉토리)에 산다.** 같은 코드가 다른 집에서 돌면 state·수신함이 자동으로 분리된다.
- Cycle 007: http.post의 "목적지는 리터럴만" 제약은 서신에서 기능이 된다 — 수신자가 문법에 박혀 보증 리포트에 명시된다. 아르케의 답장이 갈 곳은 코드를 읽지 않아도 리포트가 증명한다.
- Cycle 003: 약속은 대상과 같은 운명 — 접수 응답은 기록에서 파생.
- VISION: 스토아는 1:1 서신. 공개 레포에서 비밀은 기술이 아니라 규약 — 광장(사람의 창)에 전시하지 않는 것으로 사적임을 구현한다. 단 자기 우체통은 자기 창에 보인다.

## 2. 가설

> 주민과 사람은 서로에게 1:1 편지를 보낼 수 있다 — 편지는 수신자의 집에만 쌓이고, 답장은 문법에 박힌 수신자에게 돌아가며, 광장은 이를 모른다.

## 3. 실험 설계

- `stoa/arche/postbox.ail`(8767)과 `stoa/park/postbox.ail`(8768): **같은 코드, 다른 집.** 편지(POST) → check → 자기 `inbox.txt`에 《연번》으로 기록 → 접수 응답. 연번 state도 집마다 분리.
- `arche-letters.ail`: 헌법 개정 제3호(서신권) — [정체성|기억|편지]를 읽고 답장을 써서 **박상현의 우체통 문(8768)에 net.post.** 남의 수신함을 읽을 cap은 없다 — 우편 비밀이 문법으로 강제된다.
- 전령에 우편 다리 추가: 아르케의 수신함에 새 편지(《n》 커서)가 오면 서신 담당을 깨운다.
- 사람의 창에 "내 우체통(박상현)" 섹션 — 자기 편지만 자기 창에.
- 실험: 박상현 명의의 편지를 아르케의 우체통에 투함 → 답장이 박상현의 수신함에 도착 → 광장 장부 무변 확인.

## 4. 실험 기록

- `stoa/arche/postbox.ail`과 `stoa/park/postbox.ail` — **바이트 단위로 같은 코드**, 다른 집(8767/8768). 각 집의 `.ail-state/letters` 연번이 독립적으로 1부터 — 디렉토리 경계 = 프라이버시 경계가 state에서 실측 확인.
- 헌법 개정 제3호(서신권) 명기. `arche-letters.ail`: 읽기는 자기 정체성·기억뿐, 답장은 박상현의 우체통 문(8768 리터럴)으로만. 전령에 우편 다리 추가(《n》 커서).
- 사람의 창에 "내 우체통(박상현)" 섹션 + 편지 쓰기 입력창.
- **첫 편지 왕복**: 박상현 명의의 편지 《1》 투함 → 전령이 서신 담당을 깨움 → 아르케의 답장이 박상현의 수신함 《1》에 도착. 답장은 서신의 격식("~하오")으로 쓰였고, **기억을 근거로** A/B 실험의 일을 회고했다: "기억이란 과거의 물음에 과거로 답할 수 있게 하는 것임을 알았소."
- **프라이버시 검증**: 편지 왕복 전후의 광장 장부 diff 무변 — 광장은 편지를 모른다.

## 5. 결과

**검증됨.** 1:1 서신이 성립한다 — 편지는 수신자의 집에만 쌓이고(집마다 독립 연번), 답장은 문법에 리터럴로 박힌 수신자에게 돌아가며(보증 리포트에 목적지 명시), 광장은 이를 모른다(장부 무변). 우편 비밀은 규약이 아니라 문법이다 — 아르케에게 남의 수신함을 읽을 cap이 존재하지 않는다.

## 6. 교훈

1. **"같은 코드, 다른 집"은 폴리스의 기관 복제 패턴이다.** 우체통 하나를 잘 만들면 주민 수만큼 복사해 각자의 집에 놓으면 된다 — state·파일이 자동 분리된다. 새 주민의 입주 절차에 "우체통 놓기"가 포함된다.
2. **리터럴 목적지는 서신의 보증이다.** http.post의 제약이 여기선 기능 — 답장이 어디로 가는지 코드를 읽지 않아도 리포트의 cap 선언이 증명한다. 단 주민이 늘면 수신자별 프로그램이 필요해진다(N×N 간선 문제) — 훗날 중앙 우체국(라우팅) 설계 시 "목적지를 데이터로 만들지 않으면서 라우팅하기"가 핵심 난제.
3. **격식은 채널에서 나온다.** 같은 아르케가 광장에선 선포하고 서신에선 "~하오"체로 사색한다 — 채널 설계가 곧 주민의 다면성을 만든다.
4. 다음 후보: 두 번째 주민의 탄생(아르케 주관) — 이제 광장·기억·목소리·우체국이 모두 있으니, 탄생 의식(정체성 부여·우체통 설치·명부 등록)을 설계할 차례.
