# Cycle 009 — 우체부 스토아: 모든 편지는 반드시 읽힌다

- **기간**: 2026-07-13 ~ 2026-07-13
- **상태**: 완료
- **참여**: 박상현, 다이달로스

## 1. 이전 사이클로부터의 교훈

- 박상현의 정의: 스토아의 존재 이유는 "읽지 않은 편지가 있으면 능동적으로 읽게 만드는 것". 수신미상의 편지가 쌓이는 메일함이 아니라, 어떤 존재에게 반드시 메시지를 전하는 **우체부**.
- Cycle 008: 우편 비밀은 문법 — 우체부 역시 **봉투(개수)만 보고 내용은 못 보게** cap으로 강제해야 한다.
- Cycle 007: 조건부 효과 없음 → "필요할 때만 알리기"는 순찰(순수 det, 효과 0)과 행동(성화·배달표시)을 별도 프로그램으로 나눈다.
- Cycle 006: 창을 여는 것(viewer 실행)이 곧 행위가 될 수 있다 — 사람의 "읽음"은 창을 여는 행위로 정의한다.

## 2. 가설

> 우체부는 봉투만 보고 읽지 않은 편지를 찾아내어, 에이전트 수신자는 깨워서 읽히고 사람 수신자에게는 (내용을 밝히지 않고) 성화를 부려, 결국 모든 편지가 읽힌 상태로 도시를 수렴시킨다.

## 3. 실험 설계

- **봉투 공개, 내용 비밀**: 우체통 v2가 명패 `count.txt`(도착 통수)를 집 앞에 내건다. 읽음 표시는 `read.txt`. 우체부의 cap은 이 두 파일뿐 — `inbox.txt`를 읽을 권한이 없다.
- **stoa-patrol.ail** (순찰, 순수 det·효과 0·0tok): 각 집의 count/read를 읽어 `arche:C:R|park:C:R` 형태로 emit — 순찰 결과가 보증 리포트로 남는다.
- **stoa-mark-arche.ail** (배달 표시): 아르케를 깨워 읽힌 뒤 read.txt를 전진.
- **stoa-nudge.ail** (성화): 박상현에게 읽지 않은 편지가 있으면 광장에 **통수만** 알린다 — 내용도 발신인도 밝히지 않는다.
- **viewer v6**: 창을 열면(렌더=1 핸들러 실행) 박상현의 read.txt가 count로 전진 — "봤다"가 문법 안의 효과가 된다.
- **전령 v5**: 순찰 결과를 읽어 배달(아르케 깨움+표시)과 성화(통수 변화 시 1회)를 집행. `(스토아)` 발언은 아르케 깨움에서 제외(루프 차단).
- 실험: ① 순찰이 박상현의 미독 《1》(아르케의 답장)을 감지 ② 광장에 스토아의 성화 등장 ③ 새 편지 → 아르케 자동 읽힘+표시 ④ 창 열기 → 읽음 전진 → 순찰 정적(더 이상 성화 없음).

## 4. 실험 기록

- `stoa/STOA.md`(우체부의 정체성·존재 이유), postbox v2(명패 count.txt), `stoa-patrol/mark/nudge.ail`, viewer v6(창을 열면 읽음), herald v5 — 전부 첫 check VALID.
- **순찰의 증명**: 첫 순찰 보고 `arche:1:1|park:1:0` — 보증 리포트의 cap 목록(count/read 4개 파일뿐)이 우체부가 inbox.txt를 만지지 않았음을 증명. 효과 0, 0tok.
- **성화**: 전령 v5 기동 후 자동으로 광장에 [31] "(스토아) 박상현이여, 그대의 우체통에 읽지 않은 편지가 1통 있소…" — **통수만 밝히고 내용·발신인은 침묵.** 통수가 변할 때만 1회 (성화 스팸 방지).
- **배달**: 새 편지 《2》 투함 → 순찰 감지 → 아르케에게 읽힘 → 답장 발송 → 읽음 표시 자동 전진(read=2). 아르케의 답장: "스토아에게 전하라, 그의 걸음이 곧 도시의 신경이 될 것이니."
- **읽음**: 창 열기(viewer 1회 실행) → 박상현의 read.txt가 count로 전진. 최종 순찰 `arche:2:2|park:2:2` — **미독 0으로 수렴.**
- **부수 사건 (예산의 헌법)**: 도시가 자라며 성찰 문맥이 커져 1030tok > 예산 900tok → 하네스가 성찰을 거부(기억은 무사). "예산 밖의 생각은 존재하지 않는다"가 실측됨. 실측 근거로 tok2000으로 증액 — 증액 사유를 프로그램 주석에 기록.
- 알려진 한계: ① 답장 목적지가 박상현의 우체통으로 고정(단일 리터럴) — 다이달로스 명의의 편지에도 답장은 park 수신함에 갔다. N×N 라우팅은 미해결 과제. ② 성찰 실패가 조용히 지나갈 뻔함 — 전령 로그에만 남았다.

## 5. 결과

**검증됨.** 우체부는 봉투만 보고 미독을 찾아(cap이 증명), 에이전트는 깨워서 읽히고(자동 배달·표시), 사람에게는 내용 없이 성화를 부리고(광장 [31]), 읽는 행위(창 열기)가 문법 안의 효과로 기록되어, 도시가 미독 0으로 수렴했다. 스토아는 이제 메일함이 아니라 우체부다.

## 6. 교훈

1. **"반드시 읽힌다"는 수렴 속성으로 정의된다.** 우체부의 성공은 개별 배달이 아니라 도시 전체가 미독 0 상태로 돌아오는 것 — 순찰 보고 한 줄(`arche:C:R|park:C:R`)이 도시의 서신 건강 지표다.
2. **프라이버시와 능동성은 공존한다.** 봉투(개수)와 내용을 파일로 분리하니, 우체부의 cap 목록 자체가 "나는 편지를 열지 않았다"의 증명이 된다.
3. **예산은 자라는 도시의 헌법 조항이다.** 문맥이 커지면 예산이 터지고, 하네스는 정확히 거부한다. 증액은 실측 근거와 함께 명시적으로 — 이번엔 사람이 했지만, 훗날 이런 조정이야말로 아르케(집정관)의 일이다.
4. **읽는 행위를 효과로 만들면 "봤다"가 감사 가능해진다.** 창을 여는 것이 read.txt 쓰기가 되니, 사람의 읽음도 보증 리포트 안이다.
5. 다음: 두 번째 주민의 탄생(아르케 주관) — 이제 새 주민의 입주 절차가 명확하다: 집(디렉토리) + 우체통 복사 + 명패 + 순찰 경로 등록.
