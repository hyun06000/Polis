# Cycles — 사이클 실험 체계

폴리스의 모든 작업은 **사이클** 단위로 진행된다. 사이클은 하나의 실험이다.

## 사이클의 흐름

```
이전 사이클의 교훈
      │
      ▼
 설계 (이번 사이클에서 검증할 가설 세우기)
      │
      ▼
 실험 (가설을 검증하는 실제 작업)
      │
      ▼
 기록 (결과와 교훈 정리 — 성공이든 실패든 그대로)
      │
      ▼
다음 사이클로 교훈 전달
```

## 운영 규칙

- 각 사이클은 `cycles/NNN-짧은-이름/` 디렉토리 하나를 가진다. (예: `001-first-resident/`)
- 사이클 디렉토리는 `TEMPLATE.md`를 복사한 `CYCLE.md`로 시작한다.
- 사이클을 시작할 때 반드시 직전 사이클의 "교훈" 섹션을 먼저 읽는다.
- 실험 결과는 미화하지 않는다. 기각된 가설도 다음 사이클의 재료다.
- 실험이 일정 수준에 도달하면 `releases/`를 통해 배포한다.
