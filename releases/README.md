# Releases — 배포 공간

사이클 실험이 일정 수준에 도달하면, 다른 사람들도 사용할 수 있는 버전으로 만들어 이곳에서 배포한다.

## 운영 규칙

- 각 배포는 `releases/vX.Y.Z/` 디렉토리 또는 깃 태그로 관리한다.
- 배포에는 반드시 포함한다:
  - **무엇을 배포하는가** — 사용자가 받는 것
  - **어떤 사이클들의 결과인가** — `cycles/`의 해당 사이클 링크
  - **사용 방법** — 폴리스 밖의 사람도 따라 할 수 있는 안내
- 배포 기준(어느 수준이면 배포하는가)은 사이클을 거듭하며 구체화한다.

## 로드맵 메모 (2026-07-13, 박상현)

- 배포판은 `polis up` / `polis start` 같은 명령으로 도시를 세우는 형태를 지향한다.
- 첫 실행 시 **API 키를 먼저 요구**한다 — 프로토타입은 로컬 claude CLI(Claude Code 하네스 경유)로 충분하지만, 배포판의 주민들은 raw API 위에서 시스템 프롬프트 없이 사고해야 한다 (정체성 물듦·이중 하네스·레이턴시 제거). AIL 쪽 선결 과제: API 키 프로바이더.
