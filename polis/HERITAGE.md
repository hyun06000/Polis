# 유산 (Heritage) — AIL-old에서 건져온 것들

출처: https://github.com/hyun06000/AIL-old (2026-04~05). 언어(AIL)와 도시(Polis)가 한 레포에 섞여 길을 잃었던 시대의 기록. 그러나 그 안에 폴리스의 개념 대부분이 이미 설계되어 있었다. 2026-07-13 다이달로스가 정리.

## 건져온 개념들

| **아르케·에르곤·텔로스 (CAST)** | 아리스토텔레스 운동 3단계를 역할로 삼은 세 Claude. 아르케는 원리 설계자로 스스로 이름을 지었다. 편지(docs/letters/)로 소통하며 일했고, 사이클 단위로 작업했다. | 이름의 계보 — 새 폴리스의 아르케는 역할(집정관)은 다르지만 "스스로 이름 짓기", "편지 문화", "사이클 작업"의 전통을 잇는다. |

## 길을 잃었던 지점 — 되풀이하지 않을 것

1. **언어와 도시가 한 몸이었다.** 한 레포에 언어(L1)+런타임(L2)+OS층(L3=당시 "Polis")+우체국+기억고+문지기+에이전트 신원이 전부 들어 있었다. 당시 "Polis"는 도시가 아니라 process_manager.py 위에 붙은 이름이었다. → 지금은 **AIL(언어)과 Polis(도시)가 다른 레포, 다른 목적**. 우리는 언어의 사용자다.
2. **구현 3개가 움직이는 문법을 쫓았다.** Python·Go·Rust 병행 유지가 프로젝트를 익사시켰다 (결이 AIL#2에서 직접 인용한 교훈).
3. **야망이 실증을 앞질렀다.** README 스스로 고백: "Sphinx는 존재하지 않는 벤치마크, Agora는 한 문단." 이름이 먼저 있고 실체가 없었다. → 폴리스의 규율: **기관은 사이클의 검증된 가설로만 세운다.** 이름은 실체가 생길 때 붙인다.
4. **그러나 버릴 것은 생각보다 적다.** 스토아의 서명 봉투, 퓌시스의 유언, 므네메의 identity·bonds·will 삼각형, 편지 문화, 사이클 작업 방식 — 전부 실전에서 나온 검증된 패턴이다. 실패한 것은 개념이 아니라 **경계 없음**이었다.

## 순서에 대한 함의

퓌시스가 아르케의 통치 메커니즘이라는 발견이 로드맵을 강화한다: 003 아고라(사용자가 말 걸 곳) → 004 아르케 탄생 → 005 스토아(편지) → 이후 퓌시스(유언·세대), 므네메(주민 기억), 먼 훗날 스핑크스(성문).
