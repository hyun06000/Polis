# 2. 실험 설계 — 기대 결과를 먼저 고정한다

## 구성

| 부품 | 변경 | 역할 |
|------|------|------|
| `polis/warden.sh` v2 | 확장 | 기관 명단(이름:핸들러:포트) 전체를 기른다. 죽음 감지 → 검시 → 재기동 → 광장에 부활 공지(정문 POST) |
| `polis/necropsy.ail` | 유지 | 광장 전용 검시(파라미터 유전 포함) — 기존 퓌시스 그대로 |
| `polis/necropsy-city.ail` | 신설 | 범용 검시: 정황(기관명·exit·로그, untrusted input) → check → `city-testaments.log` 누적 + `city-testament.txt` 최신 갱신 (latest-wins + 역사, 므네메 원리). 파라미터 유전은 없음 — 기록만 |
| 기관 명단 | 신설 | viewer:8765, agora:8766, postbox-arche:8767, postbox-park:8768 |

## 기대 결과 (검증 전에 고정)

각 기관 X에 대해 `kill -9` 시:

1. 파수꾼 로그에 15초 내 `[warden] X 사망 감지 (exit=137)` 출현.
2. `city-testaments.log`에 X의 유서가 **추가**되고(누적), `city-testament.txt`가 최신으로 교체된다. 유서 기록의 보증 리포트가 로그에 남는다 (cap: 유서 파일 읽기1·쓰기2뿐, LLM 0회, 0tok). 광장은 기존 necropsy.ail 경로(유서+파라미터 유전).
3. X의 포트가 다시 응답한다 (200/403 무엇이든 — 응답 자체가 생존의 증거).
4. 광장 장부에 `(파수꾼) X 부활 …` 공지가 정문 check를 거쳐 남는다.
5. 정합성: 살해 전후 `stoa-patrol` 보고 동일, 수신함·장부 내용 무변 (부활 공지 제외).

## 범위 밖 (정직하게)

- 파수꾼 자신의 죽음 (감시자의 감시자) — 다음 사이클 후보.
- 기관별 파라미터 유전(광장 외) — 범용 검시는 기록만 한다. 유전이 필요해지는 순간 그 기관 전용 검시를 만든다.
