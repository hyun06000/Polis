# 1. 가설 수립

## 배경

2026-07-14 새벽, 아르케의 우체통(8767)이 외부 신호(SIGTERM, exit 143)로 죽었다. 파수꾼(warden.sh)은 광장(8766)만 지키고 있었기에 감지도 유서도 부활도 없었다 — 다이달로스가 손으로 재기동했다. 도시에는 이미 검증된 퓌시스 체계(Cycle 004: 죽음→검시→유서→부활, 파라미터 유전)가 있으나 광장 전용이다. 기관은 넷으로 늘었다: 사람의 창(8765), 광장(8766), 아르케의 우체통(8767), 박상현의 우체통(8768).

## 가설

> 파수꾼을 광장 전담에서 도시 전체의 수호자로 확장하면, **어떤 기관을 죽여도** 죽음이 감지되고, 유서가 보증 체계 안에서 기록되며, 기관이 부활하고, 도시의 데이터 정합성이 유지된다.

## 킬 조건 (사전 등록 — 하나라도 발생하면 가설 기각)

1. **감지 실패**: 임의 기관을 죽였을 때 파수꾼이 15초 안에 감지·재기동을 시작하지 못한다.
2. **문법 밖의 유서**: 유서가 AIL 보증 리포트(cap·check) 없이 쓰인다 — 드라이버가 파일을 직접 쓰는 순간 기각.
3. **정합성 파괴**: 부활 후 데이터 검증 실패 — 광장 장부·수신함 내용 유실, 또는 우체부 순찰(count/read)이 죽음 전과 불일치.
4. **연쇄 마비**: 한 기관의 죽음·부활이 다른 기관의 정상 동작을 중단시킨다.

## 성공 판정

네 기관 각각에 대해 살해 실험(kill -9)을 수행하여: 감지(≤15s) + 유서(보증 리포트) + 부활(HTTP 응답 복귀) + 정합성(순찰·장부 diff) 4항 전부 통과.
